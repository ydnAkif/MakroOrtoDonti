## Kritik Bulgular (Oncelik Sirasi)

1. **Yuksek - Hasta listesi id uyumsuzlugu riski (Party id -> Patient route)**
   - Hasta listesi `Party` kayitlariyla uretiliyor, ancak detay linki `patients.detail_patient` route'una `party.id` gonderiyor.
   - Bu durum id'ler birebir eslesmezse yanlis kayda gitme veya 404/500 davranisina neden olabilir.
   - Etkilenen dosya: `app/templates/patients/list.html`

2. **Yuksek - SMTP sifresi plaintext saklaniyor ve tekrar ekrana basiliyor**
   - `smtp_password` ayari veritabaninda acik metin saklaniyor.
   - Ayarlar ekraninda password alanina mevcut deger tekrar yazdiriliyor (value attr).
   - Etkilenen dosya: `app/templates/settings/index.html`

3. **Yuksek - Ortam ayarlarinda zayif varsayimlar production'a sizabilir**
   - Gelistirme `.env` degeri zayif admin sifresi + debug acik + statik secret key kombinasyonu iceriyor.
   - Etkilenen dosyalar: `.env`, `app/config.py`, `run.py`

4. **Orta - Giris endpoint'inde rate limiting / lockout yok**
   - Brute-force denemelerine karsi koruma katmani bulunmuyor.
   - Etkilenen dosya: `app/routes/auth.py`

5. **Orta - Coklu route'ta validation eksigi 500 uretebilir**
   - `date.fromisoformat`, enum cast ve float parse islemleri guard edilmeden kullaniliyor.
   - Malformed input'ta 400 yerine 500 olusabilir.
   - Etkilenen dosyalar: `app/routes/settings.py`, `app/routes/payments.py`, `app/routes/parties.py`, `app/routes/invoices.py`, `app/routes/patients.py`

6. **Orta - Dashboard/Raporlar halen Patient tabanli metrik kullaniyor**
   - Party-only akislar arttikca istatistikler eksik/yanli olabilir.
   - Etkilenen dosyalar: `app/routes/dashboard.py`, `app/routes/reports.py`

7. **Orta - Request icinde gunluk kur cekme ilk istegi yavaslatabilir**
   - `before_request` icinde dis API cagrisi, ilk istek gecikmesini artirabilir.
   - Etkilenen dosyalar: `app/__init__.py`, `app/services/exchange_service.py`

8. **Dusuk-Orta - Party-Patient cift model kaynakli veri drift riski**
   - Bazi ekranlar `Party`, bazilari `Patient` uzerinden calisiyor; tam gecis bitmeden cift yazim riski suruyor.
   - Etkilenen dosyalar: `app/routes/parties.py`, `app/routes/patients.py`

## Oncelikli Iyilestirme Plani

### 0-3 gun

1. `patients/list.html` linkini Party->Patient map ile duzelt.
2. SMTP sifresi icin asagidaki stratejiden birini uygula:
   - DB'de sifreyi encrypt et (uygulama key'i ile),
   - veya sifreyi sadece env/secret manager'da tut, UI'da asla geri gosterme.
3. Production startup policy:
   - `FLASK_DEBUG=false`
   - guclu `SECRET_KEY`
   - `DEFAULT_ADMIN_PASSWORD` kaldir veya tek kullanimlik sifreye zorla.

### 1-2 hafta

1. Login endpoint'e rate limit + gecikmeli lockout ekle.
2. Tum form parse noktalarina defensive validation ekle (400 + flash).
3. Dashboard/Reports metriklerini Party-first hale getir.
4. Rate fetch'i request-disina tasi (scheduler/background job).

### 2-4 hafta

2. Backup + restore senaryosunu otomatiklestir.
3. Guvenlik testleri ekle (CSRF enabled test seti, invalid payload testleri, auth brute-force smoke).
